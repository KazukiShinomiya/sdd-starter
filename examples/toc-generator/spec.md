## 1. 概要

Markdown ファイルの見出しを読み取り、目次（TOC）を生成して所定の位置に挿入・更新する
コマンドラインツール。外部依存を持たず、標準ライブラリだけで動く。

## 2. なぜ（背景・課題）

- **解決したい課題**: 長い Markdown 文書では目次を手で保守するのが面倒で、見出し変更と
  ずれていく。
- **対象ユーザー**: README や技術文書を Markdown で書く開発者。
- **価値**: 見出しと常に一致した目次を、ワンコマンドで保てる。

## 3. ユーザーストーリー

- **US-1**: 書き手として、Markdown の見出しから目次を生成したい（手作業をなくすため）。
- **US-2**: 書き手として、見出しを変えたら目次を最新化したい（ずれを防ぐため）。

### US-1 の受け入れ基準
- [ ] **AC-1.1**: `## 見出し` 形式の行から、階層を保った目次（ネストしたリンク一覧）が生成される。
- [ ] **AC-1.2**: 目次の各項目は、対応する見出しへのアンカーリンクになっている。
- [ ] **AC-1.3**: コードブロック内の `#` で始まる行は見出しとして扱わない。

### US-2 の受け入れ基準
- [ ] **AC-2.1**: ファイル内に目次マーカー（`<!-- TOC -->` 〜 `<!-- /TOC -->`）があれば、
      その間だけを最新の目次で置き換える（マーカー外は変更しない）。
- [ ] **AC-2.2**: マーカーが無い場合は、目次を生成して標準出力に出す（ファイルは変更しない）。

## 4. スコープ

### 含むもの（In Scope）
- 単一ファイルの見出し抽出と目次生成
- マーカー間の目次の挿入・更新
- 見出しレベル（H1〜H6）の階層保持

### 含まないもの（Out of Scope）
- 複数ファイルの一括処理
- HTML など Markdown 以外の出力形式
- 見出し以外（表・画像）への対応

## 5. 非機能要件

- **ゼロ依存**: 言語標準ライブラリのみで動作すること（このテンプレートの精神に倣う）。

## 6. 未決定事項 / 要明確化

> `/clarify` で解消済み。記録として残す。
- ~~[NEEDS CLARIFICATION: アンカーの生成規則は?]~~ → **解決**: 見出しを小文字化し、
  英数字以外をハイフンに、連続ハイフンを1つに畳む（GitHub 互換）。

## 7. 憲法との整合

- [x] この仕様は憲法（この例では「単純さ優先」「明示性」を前提）に違反しない
- 関連する条項: 単純さ優先（標準ライブラリのみ）/ 明示性（マーカー外は触らない＝挙動が予測可能）
