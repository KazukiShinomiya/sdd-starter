## 1. 概要

長いURLを短い識別子付きのURLに変換し、その短縮URLへのアクセスを元のURLへ
リダイレクトするサービス。登録者は各短縮URLのアクセス回数を確認できる。

## 2. なぜ（背景・課題）

- **解決したい課題**: 長いURLはSNSや印刷物で共有しづらく、折り返しや誤入力を招く。
- **対象ユーザー**: リンクを共有したい個人・小規模サイト運営者。
- **価値**: 短く扱いやすいリンクと、簡易なアクセス計測を得られる。

## 3. ユーザーストーリー

- **US-1**: 利用者として、長いURLを短縮URLに変換したい（共有を容易にするため）。
- **US-2**: 閲覧者として、短縮URLにアクセスして元のページに到達したい。
- **US-3**: 登録者として、自分の短縮URLのアクセス回数を確認したい。

### US-1 の受け入れ基準
- [ ] **AC-1.1**: 有効なURLを入力すると、一意な短縮識別子を含むURLが返る。
- [ ] **AC-1.2**: 不正な形式のURLを入力すると、作成は拒否されエラーが返る。
- [ ] **AC-1.3**: 同一URLを複数回登録すると、それぞれ別の識別子が発行される。

### US-2 の受け入れ基準
- [ ] **AC-2.1**: 存在する短縮URLにアクセスすると、元URLへHTTPリダイレクトされる。
- [ ] **AC-2.2**: 存在しない識別子にアクセスすると、404が返る。

### US-3 の受け入れ基準
- [ ] **AC-3.1**: 識別子を指定すると、累計アクセス回数が取得できる。
- [ ] **AC-3.2**: 一度もアクセスされていない識別子のアクセス回数は0と返る。

## 4. スコープ

### 含むもの（In Scope）
- URLの登録・短縮識別子の発行
- 短縮URLから元URLへのリダイレクト
- 識別子ごとのアクセス回数集計と取得

### 含まないもの（Out of Scope）
- ユーザー認証・アカウント管理（今回は登録者の概念を持たない単一利用を想定）
- カスタム識別子（任意の短縮文字列の指定）
- 有効期限・リンク削除
- 地理/参照元などの詳細アナリティクス

## 5. 非機能要件

- **性能**: リダイレクトの応答は95パーセンタイルで100ms以内。
- **一意性**: 識別子は衝突しないこと（再発行で既存と重複しない）。
