> **段階**: ④ Plan の補助（`plan.md` §3 の詳細）
> **対応する設計**: `./plan.md`  |  **対応する仕様**: `./spec.md`
> **作成日**: 2026-05-31
>
> ※ テンプレートの動作例。`templates/data-model-template.md` を雛形に、
> plan §3 の1行エンティティを実装に落とせる粒度へ展開した。

---

## 1. エンティティ一覧

| エンティティ | 役割（何を表すか） | 対応する仕様 |
|--------------|--------------------|--------------|
| Link | 1件の短縮URL（元URL・識別子・アクセス回数） | US-1, US-2, US-3 |

## 2. 各エンティティの定義

### Link

| 属性 | 型 | 必須 | 制約・既定値 | 説明 |
|------|----|----|--------------|------|
| id | INTEGER | ✓ | 主キー・自動採番 | 内部識別子（外部には出さない） |
| code | TEXT | ✓ | 一意・7文字 Base62 | 短縮識別子。URL に現れる |
| original_url | TEXT | ✓ | 妥当な URL 形式 | リダイレクト先（AC-1.2 で形式検証） |
| created_at | TEXT | ✓ | 既定 = 現在時刻(ISO8601) | 登録時刻 |
| hits | INTEGER | ✓ | 既定 0・非負 | 累計アクセス回数（AC-3.1, 3.2） |

> `code` の一意制約は AC-1.1（一意な識別子）と非機能要件（衝突しない）の直接の根拠。

## 3. 関係（リレーション）

| 関係 | 多重度 | 説明・削除時の挙動 |
|------|--------|--------------------|
| （なし） | — | 単一エンティティ。関連を持たない（スコープに認証・所有者を含まないため） |

## 4. 状態遷移

> Link は明示的な状態機械を持たない（有効期限・削除は Out of Scope）。
> 変化するのは `hits` の増加のみ。

## 5. 不変条件（Invariants）

- `code` は全 Link で一意。
- `hits` は 0 以上で、単調増加（減ることはない）。
- `original_url` は登録後に変更されない（編集機能は Out of Scope）。

> これらは実装時のバリデーションとテスト（AC-1.1 / AC-3.2）の根拠になる。
