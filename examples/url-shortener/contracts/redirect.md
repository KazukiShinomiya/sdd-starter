# 契約: リダイレクト（GET /{code}）

> **段階**: ④ Plan の補助（`plan.md` §4 の詳細）
> **対応する設計**: `./../plan.md`  |  **対応する仕様**: `./../spec.md`
> **作成日**: 2026-05-31
>
> ※ テンプレートの動作例。`templates/contract-template.md` を雛形に書いた。

---

## 1. 概要

- **種別**: HTTP API
- **責務**: 短縮識別子から元URLを解決し、リダイレクトする。あわせてアクセス回数を加算する。
- **対応する仕様**: US-2 / AC-2.1, AC-2.2（および hits 加算で US-3 を支える）

## 2. シグネチャ / エンドポイント

```
GET /{code}
```

## 3. 入力

| 名前 | 型 | 必須 | 制約 | 説明 |
|------|----|----|------|------|
| code | string（パス） | ✓ | 7文字 Base62 | 短縮識別子 |

## 4. 出力（正常系）

| フィールド | 型 | 説明 |
|------------|----|------|
| Location ヘッダ | string | 元URL |

- **成功の判定**: HTTP 302 Found（Location に元URL）。副作用として該当 Link の `hits` を +1。

## 5. エラー（異常系）

| 条件 | 返し方 | 呼び出し側の対処 |
|------|--------|------------------|
| code が存在しない | 404 Not Found | 識別子を確認（AC-2.2） |

## 6. 事前条件 / 事後条件

- **事前条件**: なし。
- **事後条件**: 解決成功時のみ `hits` が1増える（解決失敗時は加算しない）。
- **冪等性**: 解決自体は冪等だが、`hits` 加算という副作用は冪等でない（呼ぶ度に増える）。
