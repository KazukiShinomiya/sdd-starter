# 契約: リンク登録（POST /links）

> **段階**: ④ Plan の補助（`plan.md` §4 の詳細）
> **対応する設計**: `./../plan.md`  |  **対応する仕様**: `./../spec.md`
> **作成日**: 2026-05-31
>
> ※ テンプレートの動作例。`templates/contract-template.md` を雛形に書いた。

---

## 1. 概要

- **種別**: HTTP API
- **責務**: 元URLを受け取り、一意な短縮識別子を発行して保存する。
- **対応する仕様**: US-1 / AC-1.1, AC-1.2, AC-1.3

## 2. シグネチャ / エンドポイント

```
POST /links
Content-Type: application/json
```

## 3. 入力

| 名前 | 型 | 必須 | 制約 | 説明 |
|------|----|----|------|------|
| url | string | ✓ | 妥当な http(s) URL | 短縮したい元URL |

## 4. 出力（正常系）

| フィールド | 型 | 説明 |
|------------|----|------|
| code | string | 発行された7文字の識別子 |
| short_url | string | `code` を含む短縮URL |

- **成功の判定**: HTTP 201 Created

## 5. エラー（異常系）

| 条件 | 返し方 | 呼び出し側の対処 |
|------|--------|------------------|
| url が無い/形式不正 | 400 Bad Request | 入力を修正して再送（AC-1.2） |
| 識別子の生成が連続衝突 | 500（リトライ上限超過） | 時間をおいて再送 |

## 6. 事前条件 / 事後条件

- **事前条件**: なし（認証不要。スコープ外）。
- **事後条件**: Link が1件保存され、`code` は既存と重複しない（AC-1.1）。
- **冪等性**: 非冪等。同一URLの再登録は別の `code` を発行する（AC-1.3）。
