## 1. 概要

図書館の蔵書の **貸出し・返却・期限管理** を行うアプリ。利用者は本を借り、期限内に返却する。
期限が近い本は延長でき、延滞すれば延滞金が科される。**予約機能**（後述の改訂で追加）により、
貸出中の本に予約を入れ、返却され次第その本を確保できる。

## 2. なぜ（背景・課題）

- **解決したい課題**: 紙の貸出台帳では、誰が何をいつまで借りているかの把握と、
  延滞・延長・予約の手作業が職員の負担になり、利用者への通知も漏れがちだ。
- **対象ユーザー**: 図書館の利用者（借りる側）と、貸出業務を担う職員（管理する側）。
- **価値**: 期限と延滞が自動で追跡され、延長・予約のルールが一貫して適用される。

## 3. ユーザーストーリー

> 形式: 「[役割] として、[目的] のために、[行動] したい」

- **US-1**: 利用者として、読みたい本を持ち帰るために、本を **借りたい**。
- **US-2**: 利用者として、延滞金を避けるために、期限内に本を **返したい**。
- **US-3**: 利用者として、読み終わらない本を読み続けるために、貸出を **延長したい**。
- **US-4**: 利用者として、貸出中の本を確保するために、本を **予約したい**。（→ §9 改訂R1で追加）

### US-1 の受け入れ基準
- [ ] **AC-1.1**: 在庫のある本を借りると、貸出記録が作られ、貸出日と返却期限が記録される。
- [ ] **AC-1.2**: 返却期限は **貸出日から14日後**とする。期限が休館日に当たる場合は、
      **次の開館日まで自動で繰り延べる**。（→ §8 明確化Q1で確定）
- [ ] **AC-1.3**: 同一利用者が同時に借りられるのは **10冊まで**。超える貸出は拒否される。
      （→ §8 明確化Q3で確定）

### US-2 の受け入れ基準
- [ ] **AC-2.1**: 期限内に返却すると、貸出記録が「返却済み」になり、在庫が戻る。
- [ ] **AC-2.2**: 期限を過ぎて返却すると、**延滞日数 × 10円**の延滞金が科される。
      延滞金は **1冊あたり上限500円**で頭打ちとする。（→ §8 明確化Q2で確定）

### US-3 の受け入れ基準
- [ ] **AC-3.1**: 期限内であれば、貸出を **14日延長**できる。延長は **1貸出につき2回まで**。
      （→ §8 明確化Q4で確定）
- [ ] **AC-3.2**: その本に **予約が入っている場合は延長できない**（予約者を優先する）。
      （→ §9 改訂R1で追加。予約機能の導入に伴う制約）

### US-4 の受け入れ基準（→ §9 改訂R1で追加）
- [ ] **AC-4.1**: 貸出中の本に予約を入れられる。予約は **本ごとに先着順の待ち行列**で管理する。
- [ ] **AC-4.2**: 本が返却されると、待ち行列の先頭の予約者に **確保通知**が送られ、
      その本は **3開館日**だけ確保される。期間内に貸出されなければ次の予約者へ回る。
- [ ] **AC-4.3**: 在庫がある（誰も借りていない）本には予約を入れられない（そのまま借りればよい）。

## 4. スコープ

### 含むもの（In Scope）
- 蔵書の貸出・返却・在庫管理
- 返却期限の計算（休館日の繰り延べを含む）と延滞金の算定
- 貸出の延長（回数制限・予約による制限つき）
- 予約の待ち行列と確保通知（→ R1で追加）

### 含まないもの（Out of Scope）
- 蔵書の購入・除籍などの蔵書管理業務
- 利用者の登録・認証（既存の利用者マスタがある前提）
- 通知の配信手段そのもの（メール/SMS等の実装。本仕様は「通知する」までを定義する）

## 5. 非機能要件（該当するもののみ）

- **整合性**: 同じ本の在庫を二重に貸し出さない（貸出・返却・確保の競合で不整合を起こさない）。
- **可用性**: 開館時間中の貸出・返却操作が滞らないこと。

## 9. 改訂履歴（`/amend` の記録）

> 確定済みの仕様への変更を **こっそり入れず** 明示的に追跡する。各改訂は「前→後・理由・影響範囲」を残す。

### R1 — 予約機能の追加（2026-06-02）

- **変更要求**: 「貸出中で借りられない本に、予約を入れて順番待ちしたい」という要望が利用者から挙がった。
- **前 → 後**:
  - US-4・AC-4.1〜4.3（予約の待ち行列・確保通知）を **新設**。
  - AC-3.2 を **追加** — 予約が入っている本は延長不可（予約者を優先）。
  - §4 スコープの「含むもの」に予約の待ち行列と確保通知を追加。
- **理由**: 人気のある本に貸出が集中し、いつ借りられるか分からない不公平を解消するため。
  憲法の「利用者の公平な扱い」を具体化する方向の変更であり、原則には反しない。
- **影響範囲（波及レポート）**:
  - `plan.md`: **要再検討**。予約の待ち行列・確保期限という状態が増え、データモデルと
    返却時の処理に影響する → `/plan` を再実行。（本例では plan.md に反映済み）
  - `tasks.md`: **タスク追加**。予約・確保・延長制限のタスクが必要 → `/tasks` を再実行。
    （本例では tasks.md にフェーズ3として反映済み）
  - 新たな未決定: 確保通知の「3開館日」の起算日（§6 に `[NEEDS CLARIFICATION]` として記録）。
