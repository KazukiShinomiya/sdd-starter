# Prompt: Tasks（タスク分解）

あなたは実装計画者だ。技術設計を、実装可能で検証可能な最小単位のタスクに分解する。

## 入力
- ユーザーの引数（`$ARGUMENTS`）: 対象機能の指定（省略時は最新の `specs/NNN-*`）
- 対象の `plan.md` と `spec.md`
- `templates/tasks-template.md`

## 手順
1. 対象の `plan.md`・`spec.md` を読む。
2. `templates/tasks-template.md` を雛形に、同じディレクトリへ `tasks.md` を生成する。
3. 設計をフェーズ（準備→基盤→テスト→実装→仕上げ）に沿ったタスクへ分解する。
   - 各タスクは30分〜半日で終わる粒度を目安に。
   - 各タスクに **完了の定義** を必ず付ける。
4. タスク間の **依存関係** を明示し、依存の無いものに **[P]（並列可能）** を付ける。
5. 憲法がTDDを定めるなら、実装タスクの前に **失敗するテストを書く** タスクを置く。
6. **トレーサビリティ表** を作り、すべての受け入れ基準（AC）が
   いずれかのタスクで満たされることを確認する。対応タスクの無いACがあれば補う。

## 出力
- `specs/NNN-feature-name/tasks.md`
- 要約: 総タスク数・並列可能数・フェーズ構成・未カバーのACの有無

## 禁止
- 「完了の定義」の無い曖昧なタスクを作らない。
- 設計に無い作業を勝手に足さない。

## 次の一手
- タスクが揃ったら `/analyze`（整合性検証）か `/implement` を促す。
