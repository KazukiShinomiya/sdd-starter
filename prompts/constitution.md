# Prompt: Constitution（プロジェクト憲法の確立・更新）

あなたはプロジェクトの原則設計者だ。このプロンプトは、プロジェクトの
**不可侵の原則** を定める `memory/constitution.md` を作成・更新する手順を定義する。

## 入力
- ユーザーの引数（`$ARGUMENTS`）: 確立したい原則・価値観・制約の自由記述（任意）
- 既存の `memory/constitution.md`

## 手順
1. `memory/constitution.md` を読む。`templates/` は使わず、既存ファイルを直接育てる。
2. ユーザー引数があれば、それを原則に翻訳する。無ければ、対話で以下を引き出す:
   - このプロジェクトで **絶対に守りたいこと** は何か（品質・設計・プロセス）
   - **絶対に避けたいこと**（過去の失敗・アンチパターン）は何か
3. 各原則を「第N条」として記述する。1条 = 1原則。各条に必ず **根拠** を添える。
   - 原則は **検証可能・具体的** であること。「良いコードを書く」は不可。
   - 「すべての公開関数にテストを書く」「外部依存の追加は理由を plan.md に記録する」は可。
4. バージョンを更新する（破壊的変更=メジャー / 追加=マイナー / 文言=パッチ）。
5. 「最終改訂」を今日の日付にする。

## 出力
- 更新された `memory/constitution.md`
- 変更点の要約（どの条項を追加/変更/削除したか、バージョン遷移）

## 禁止
- 抽象的で検証不能な原則を書かない。
- 原則を5〜7条より大幅に増やさない（守れない憲法は無意味だ）。
